Representación de Datos y Operaciones

Los datos pueden representarse en diversos formatos dependiendo del tipo de información que se desee almacenar. Pueden ser:

  • Enteros (int): Número sin decimal
  • Punto flotante (float): Números que incluyen decimales
  • Cadenas de texto (string): Texto compuesto por caracteres
  • Booleanos(bool): Valores de verdad (verdadero o falso)

Sistemas Numéricos y Conversiones

son métodos para representar números. Podemos utilizar:

  • Decimal base 10, que usa los dígitos del 0 al 9
  • Binario base 2, utiliza solo dos dígitos, 0 y 1
  • Hexadecimal base 16, usa los dígitos del 0 al 9 y las letras de la A a la F representando 11-15

Operadores y Operandos

Un operador es un símbolo que le dice al compilador o intérprete que realice una operación matemática, lógica o de otro tipo, entre operandos. Por ejemplo, en la expresión "5" + "3", el "+" es el operador y "5" y "3" son los operandos

Prioridad de Operadores

La prioridad de operadores determina el orden en el que se realizan las operaciones en una expresión. Por ejemplo, en las multiplicaciones y divisiones tienen prioridad sobre las sumas y restas. Así que en la expresión "3 + 4 * 2", primero se realiza la multiplicación "4 * 2" y luego se suma el "3" para dar como resultado = "11".

Estructura de seleccion

permiten que un programa tome decisiones y ejecute diferentes secciones de código dependiendo de ciertas condiciones

  • If: La estructura más simple de selección. Ejecuta un bloque de código solo si se cumple una condición específica
  • If-Else: Extiende la estructura if al agregar un bloque de código alternativo que se ejecuta si la condición inicial no se cumple
  • If-Elif-Else: Permite múltiples chequeos condicionales en secuencia. Si la primera condición no se cumple, se puede evaluar otra, y así sucesivamente

Estructura de iteracion

tambien concidas como bucles, permiten repetir un bloque de código múltiples veces, generalmente hasta que se cumpla una condición o se termine de recorrer un conjunto de elementos

    For: Repite el bloque de código un número determinado de veces
    While: Continúa ejecutando el bloque de código mientras la condición sea verdadera

Implementacion de algoritmos

Es el proceso de traducir un algoritmo, que es básicamente una serie de pasos definidos para resolver un problema, a un código de programación que puede ser ejecutado por una computadora. Este proceso involucra elegir un lenguaje de programación adecuado, escribir el código de manera eficiente y asegurarse de que el algoritmo se ejecute correctamente y maneje todas las situaciones esperadas sin errores.

Recursividad y Modulos

La recursividad es un concepto en programación donde una función se llama a sí misma dentro de su propia definición. Esto es útil para resolver problemas que pueden descomponerse en subproblemas más pequeños de la misma forma que el problema original. La clave es asegurar que haya una condición de terminación para evitar que la función se llame a sí misma infinitamente.

Los módulos en programación son una manera de organizar y reutilizar el código dividiendo un programa grande en partes más pequeñas y manejables. Cada módulo contiene un grupo de funciones, clases u otros elementos relacionados que realizan tareas específicas. Los módulos ayudan a mantener el código limpio, fácil de entender y de mantener. Además, permiten a los desarrolladores compartir y usar código de otros sin necesidad de reescribirlo.

Arreglos

Es una estructura de datos que almacena una colección de elementos del mismo tipo en un bloque contiguo de memoria. Los elementos de un arreglo pueden ser accedidos directamente mediante un índice, lo que hace que los arreglos sean muy eficientes para ciertos tipos de operaciones computacionales. Los arreglos son fundamentales para muchos algoritmos porque permiten manejar fácilmente conjuntos de datos.

pruebas, depuracion, documentacion y mantenimiento

Pruebas (Testing) Las pruebas son un proceso crucial en el desarrollo de software que implica verificar que el código funcione correctamente y cumpla con los requisitos especificados. El objetivo es identificar errores, deficiencias o requisitos no cumplidos en el código.

Depuración (Debugging) La depuración es el proceso de identificar, aislar y corregir errores o bugs en el software. Esto implica:

  • Identificar el error en el código.
  • Localizar la sección específica del código donde ocurre el error.
  • Corregir el error y verificar que el código ahora funcione correctamente sin introducir otros problemas.

La documentación es esencial para cualquier proyecto de software. Proporciona una guía clara de lo que hace el software, cómo se usa, y cómo otros desarrolladores pueden trabajar con el código.

El mantenimiento de software involucra modificar y actualizar el software después de su despliegue inicial para corregir errores, mejorar el rendimiento o adaptar el software a nuevos requisitos. Gestionando la correcion de errores, la mejora y adaptacion

Instrumentacion de programas

implica insertar código adicional en un programa durante su desarrollo para recopilar datos sobre cómo se ejecuta. Este código puede medir varios aspectos, como el tiempo de ejecución de ciertas funciones, el uso de recursos del sistema (memoria, CPU), y la frecuencia de ciertas operaciones.

Tecnicas de Implementacion:

  • Instrumentación estática: Se introduce código de instrumentación en el código fuente del programa antes de su compilación. Esta técnica es útil para pruebas de rendimiento y depuración durante el desarrollo.
  • Instrumentación dinámica: Se inserta código de instrumentación en tiempo de ejecución sin modificar el código fuente. Esto es especialmente útil para sistemas en producción donde no se desea o no es posible cambiar el código fuente.
